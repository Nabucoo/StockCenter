# Documentação de Engenharia de Software

Este diretório reúne os artefatos de documentação do projeto **StockCenter**, produzidos durante a etapa de engenharia de software.

O objetivo deste material é apresentar, para leitores externos, o levantamento de requisitos e a modelagem do sistema, complementando a descrição funcional que está no `README.md` principal do repositório.

## O que está aqui

- `requisitos/` contém o documento de levantamento de requisitos do sistema.
- `diagramas/` contém os arquivos de modelagem visual usados para representar o sistema.

## Conteúdo principal

- `requisitos/Levantamento de requisitos.docx` — documento de requisitos do StockCenter.
- `diagramas/Diagrama classes` — diagrama de classes do sistema.
- `diagramas/Diagrama de caso de uso` — diagrama de casos de uso.
- `diagramas/diagrama ER.mwb` — modelo entidade-relacionamento do banco de dados.

## Para quem lê este diretório

Este material é destinado a:

- avaliadores da disciplina de Engenharia de Software;
- membros da equipe que revisam a modelagem do sistema;
- terceiros que precisam entender como o projeto foi documentado.

## Como navegar

- Comece pelo documento de requisitos para compreender os objetivos e funcionalidades do sistema.
- Em seguida, veja os diagramas para entender a estrutura e o comportamento do StockCenter.

> Dica: para facilitar a visualização no GitHub, mantenha cópias exportadas dos diagramas em formatos como PNG ou PDF.
